# Santa Fe CI — Skeleton Integration Change Summary

## Overview

The MegaCap 50 dashboard skeleton (a financial data visualization template) was integrated into the Santa Fe CI real estate competitive intelligence platform. The skeleton's architecture, layout system, and visual design tokens were adopted while preserving all backend functionality.

## Files Changed

### New Files Created (in `/frontend/`)

| File | Size | Description |
|------|------|-------------|
| `frontend/index.html` | ~5KB | HTML shell adapted from skeleton. Contains sections for overview (KPIs, buildings, events, listings), detail view (scores, negotiation band, intel grid, inquiry form), compare view, and agents view. |
| `frontend/style.css` | ~30KB | Merged CSS: skeleton design tokens (type scale, spacing, colors, radius, shadows) + Santa Fe CI components (building color-coding, score gauges, negotiation bands, battle cards, counterparty playbook, inquiry forms, agents table). |
| `frontend/base.css` | ~1KB | CSS reset copied from skeleton. |
| `frontend/app.js` | ~33KB | JavaScript using skeleton's architecture (view management, Chart.js integration, theme toggle) wired to Santa Fe CI API endpoints (`/api/feed`, `/api/agents`, `/api/inquiries`, `/api/refresh`). |

### Data Files Created (in `/data/`)

| File | Description |
|------|-------------|
| `data/listings.seed.json` | 14 seed listings across 3 buildings (Peninsula, Torre 300, Paradox) |
| `data/listings.live.json` | Auto-generated by `scraper.py` from seed data with enriched intel |

### Backend Files — Unchanged

All Python backend files remain untouched:
- `server.py` — HTTP server serving `/frontend/` directory + all `/api/*` routes
- `intelligence.py` — Scoring engine v2, negotiation bands, battle cards
- `scraper.py` — Merges seed → live data
- `collector.py` — Collection/tracking engine
- `operator_engine.py` — Operator actions
- `scoring_engine_v2.py` — Quantitative scoring

## Design Decisions

| Decision | Rationale |
|---|---|
| Building accent colors: teal (Peninsula), amber (Torre 300), purple (Paradox) | Distinct, accessible, work in both light/dark mode |
| Spanish UI labels | Santa Fe CDMX context, matches the real estate market |
| Score color-coding: ≥70 green, ≥40 amber, <40 red (inverse for ghost) | Standard traffic-light scoring pattern |
| Counterparty playbook with collapsible tactics | Rich data (8+ tactics per listing) needs progressive disclosure |
| Battle card as label:value split on colon | Data comes as "Fair band: MXN 63,000–68,000" — splitting gives clean layout |
| `data-theme` attribute for dark mode | Matches skeleton pattern, CSS variables handle all theme-dependent values |

## Bug Fixes Applied

1. **battle_card rendering**: Data is `string[]` like `["Fair band: MXN 63,000–68,000", ...]`. Fixed to split on first colon into label/value pairs with monospace values.

2. **script rendering**: Data is a plain string (negotiation script paragraph), not `{opening, body}` object. Fixed to render as full paragraph with `pre-line` whitespace.

3. **counterparty_playbook rendering**: Data has `primary_tactic` (object with name, description, tell, rebuttal_script, say_instead, do_not_say, probability, evidence) + `tactics[]` array + `market_context_note` + `counter_script`. Fixed to render:
   - Primary tactic highlighted with badge + probability
   - Color-coded fields (green = "say instead", red = "do not say")
   - Market context and counter script blocks
   - Collapsible "Más tácticas (N)" for remaining tactics

## Running Locally

```bash
cd /home/user/workspace
python3 server.py
# Open http://localhost:8000/
```

The frontend is served directly by the Python server from the `frontend/` directory. All API calls use relative paths (`/api/feed`, etc.) — no separate frontend server needed.

## Deployment Note

The deployed static version shows the UI shell but requires the Python server for data. When running locally on port 8000, all views function fully: overview with KPIs/buildings/listings, detail with scores/negotiation/intel, compare charts, and agents directory.
